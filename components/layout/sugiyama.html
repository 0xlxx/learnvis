Lesson 0001 — Sugiyama 框架：层次布局的五大阶段

前置知识：DAG、拓扑排序、BFS/DFS · 关联课程：无（此为第一课）· 预计完成时间：15 分钟

你要解决什么问题？

假设你有一个有向图——比如函数调用链、类继承关系、CI/CD 流水线——你 想让节点按层次排列，边尽量朝一个方向流动。这不是随便拖拽能做到的，需要算法。

Sugiyama 框架（Sugiyama, Tagawa & Toda, 1981）是解决这个问题的经典方案。 它是 ELK 层次布局引擎的核心，也是几乎所有层次布局算法的基础。它把问题拆成 5 个连续阶段：

阶段名称输入输出核心问题
P1 消环 Cycle Breaking 任意有向图 无环图（DAG） 哪些边需要反转？
P2 分层 Layer Assignment DAG 每节点一个层级号 每个节点放哪一层？
P3 减少交叉 Crossing Minimization 分层节点 每层节点排好序 怎么排节点让交叉最少？
P4 节点放置 Node Placement 有序分层 节点 (x, y) 坐标 节点放在什么位置让边尽量直？
P5 边路由 Edge Routing 节点坐标 + 边 边的路径 边怎么画才能不穿过节点？
关键洞察：这 5 个阶段是串行管道——每个阶段的输出是下一阶段的输入。 这也是为什么你可以独立优化每个阶段，而不需要同时处理所有约束。 这是一个 divide and conquer 策略的经典案例。

P1: 消环

观察要点：P1 反转了一条边 → P2 分配层级 → P3 调整 A 和 B 的上下顺序减少交叉 → P4 放置坐标 → P5 画出正交边

为什么必须是这个顺序？

顺序不可调换，因为每一步依赖上一步的结果：

  • P1→P2：如果不先消环，分层算法根本无法处理——在有环图上"上游/下游"没有意义
  • P2→P3：必须先知道每个节点属于哪一层，才能在层内调整顺序
  • P3→P4：必须先固定节点在层内的相对顺序，再分配精确坐标
  • P4→P5：边路由需要知道节点的精确位置才能避开它们
记住这个框架。接下来的每一课都会深入其中一个阶段。 理解这 5 阶段的"输入→输出"是你实现整个引擎所需的心理模型。

注意：这些阶段中的大多数问题是 NP-hard 的。 例如，Crossing Minimization（P3）即使是在 2 层之间优化节点顺序也是 NP-hard 的。 我们实际使用的是启发式算法——在合理时间内找到"足够好"的解，而非最优解。 这是整个领域的核心张力：美感目标 vs 计算可行性。
